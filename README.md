# แดชบอร์ดข่าวกองทัพบก

เว็บแดชบอร์ด static (ไม่มีเซิร์ฟเวอร์) อ่าน/ค้นหา/ส่งออกข้อมูลข่าวจากระบบ Google Alert กองทัพบก
รองรับทั้งคอมพิวเตอร์และมือถือ อัปเดตข้อมูลอัตโนมัติทุก 3 ชั่วโมงผ่าน GitHub Actions
**ไม่แตะ/แก้ไขข้อมูลในชีตต้นทางเลย — อ่านอย่างเดียวผ่านลิงก์ Publish to web**

## สถาปัตยกรรม

```
Google Sheet --(Publish to web CSV, อ่านอย่างเดียว)--> GitHub Actions (ทุก 3 ชม.)
    --> data/news.json --> GitHub Pages (เว็บ static) --> ผู้ใช้ (คอม/มือถือ)
```

## โครงสร้างไฟล์

```
index.html                       หน้าเว็บหลัก
style.css                        สไตล์ responsive
app.js                           โหลดข้อมูล + กรอง/ค้นหา + ส่งออก CSV
data/news.json                   ข้อมูลข่าว (GitHub Actions เขียนทับอัตโนมัติทุก 3 ชม.)
scripts/fetch-and-build.js       สคริปต์ดึง CSV จากชีต แล้วแปลงเป็น news.json
.github/workflows/update-data.yml   GitHub Actions workflow (cron ทุก 3 ชม.)
package.json                     รายการไลบรารีที่สคริปต์ build ใช้
```

## ขั้นตอนติดตั้ง (ทำครั้งเดียว)

### 1. Publish ชีตเป็น CSV (อ่านอย่างเดียว ไม่กระทบข้อมูลต้นฉบับ)

1. เปิด Google Sheet "สรุปgoogle alert"
2. File > Share > **Publish to web**
3. เลือกชีตที่ต้องการ (แผ่นงานเดียว ไม่ใช่ทั้งไฟล์) เลือกรูปแบบ **Comma-separated values (.csv)**
4. กด Publish แล้วคัดลอกลิงก์ที่ได้ (ขึ้นต้นด้วย `https://docs.google.com/spreadsheets/d/e/.../pub?...`)

> ขั้นตอนนี้เปิดเผยเฉพาะข้อมูลในชีตผ่านลิงก์ CSV เท่านั้น ไม่ได้แก้ไขสิทธิ์การเข้าถึงไฟล์ชีตจริง และไม่มีการเขียนข้อมูลกลับเข้าไปเลย

### 2. สร้าง GitHub repository

1. สร้าง repo ใหม่บน GitHub (public)
2. อัปโหลดไฟล์ทั้งหมดในโฟลเดอร์นี้ขึ้น repo (รวมโฟลเดอร์ `.github/` ด้วย — ถ้าใช้เว็บ GitHub อัปโหลด ต้องเปิดให้เห็นไฟล์ที่ขึ้นต้นด้วยจุดก่อน หรือใช้ `git push` จากเครื่องจะสะดวกกว่า)

### 3. ตั้งค่า Secret สำหรับลิงก์ CSV

1. ใน repo ไปที่ Settings > Secrets and variables > Actions > New repository secret
2. ชื่อ: `SHEET_CSV_URL`
3. ค่า: ลิงก์ CSV ที่คัดลอกจากขั้นตอนที่ 1

### 4. เปิดสิทธิ์ให้ Actions commit กลับ repo ได้

1. Settings > Actions > General > เลื่อนลงไปที่ **Workflow permissions**
2. เลือก **Read and write permissions**
3. Save

### 5. เปิดใช้งาน GitHub Pages

1. Settings > Pages
2. Source: Deploy from a branch
3. เลือก branch `main` และโฟลเดอร์ `/ (root)`
4. รอสักครู่ จะได้ลิงก์เว็บ เช่น `https://<username>.github.io/<repo-name>/`

### 6. รันรอบแรกด้วยมือ (ไม่ต้องรอ 3 ชม.)

1. ไปที่แท็บ **Actions** ของ repo
2. เลือก workflow "Update news data" ทางซ้าย
3. กด **Run workflow** เพื่อรันทันที
4. รอสักครู่ ถ้าสำเร็จจะเห็นไฟล์ `data/news.json` ถูก commit อัตโนมัติ พร้อมข้อมูลข่าวจริง

หลังจากนี้ระบบจะรันอัตโนมัติทุก 3 ชั่วโมง (เวลา UTC `0 */3 * * *` = เวลาไทย 07:00/10:00/13:00/16:00/19:00/22:00/01:00/04:00 น.)

## ฟีเจอร์ในแดชบอร์ด

- **ค้นหา**: พิมพ์คำค้นในหัวข้อ/เนื้อหาย่อ/แหล่งที่มา
- **กรอง**: ตามหมวดข่าว, ช่วงวันที่, เฉพาะข่าวลบ, เฉพาะไทย-กัมพูชา
- **เรียงลำดับ**: ใหม่ล่าสุดก่อน / เก่าสุดก่อน / นำเสนอมากสุดก่อน
- **ส่งออก CSV**: ส่งออกเฉพาะข่าวที่ตรงตัวกรองปัจจุบัน เปิดใน Excel ได้ทันที (รองรับภาษาไทย)
- **จัดกลุ่มข่าวอัตโนมัติ**: ข่าวเรื่องเดียวกันจากหลายสำนักถูกรวมเป็น "ประเด็น" เดียว ตามรหัสกลุ่มข่าวเดียวกับระบบ LINE
- **Responsive**: ตารางการ์ดปรับคอลัมน์ตามขนาดจอ (มือถือ 1 คอลัมน์ / แท็บเล็ต 2 คอลัมน์ / คอมพิวเตอร์ 3 คอลัมน์)

## หมายเหตุ

- ระบบนี้เป็นคนละส่วนกับระบบแจ้งเตือน LINE (ไฟล์ `.gs`) โดยสมบูรณ์ — ใช้ข้อมูลจากชีตเดียวกัน แต่ดึงผ่าน CSV แยกต่างหาก ไม่แตะโค้ด Apps Script เลย
- ข้อมูลบนเว็บอาจ "ช้ากว่าชีตจริง" สูงสุดประมาณ 3 ชั่วโมง ตามรอบการอัปเดต
- เว็บนี้เปิดเผยต่อสาธารณะ (ไม่มีระบบล็อกอิน) ตามที่ตกลงกันไว้
